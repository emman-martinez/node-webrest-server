# node-webrest-server

Setup guide to run the project from scratch (after cloning the repository).

## Requirements

- Node.js 20+
- npm
- Docker + Docker Compose

## 1) Clone and install dependencies

```bash
git clone <URL_DEL_REPO>
cd restweb
npm install
```

## 2) Environment variables

Create `.env` from the template:

```bash
cp .env.template .env
```

Expected default values:

```env
PORT=3000
PUBLIC_PATH=public

POSTGRES_URL=postgresql://postgres:123456@localhost:5432/TODO
POSTGRES_USER=postgres
POSTGRES_DB=TODO
POSTGRES_PORT=5432
POSTGRES_PASSWORD=123456

NODE_ENV=development
```

## 3) Start PostgreSQL with Docker

```bash
docker compose up -d
docker compose ps
```

You should see the `postgres-db` service in `Up` status.

## 4) Create Prisma files manually (first time in a clean clone)

If the repo does not include `prisma/` or `prisma.config.ts`, create them with these commands.

Initialize Prisma:

```bash
npx prisma init
```

Create `prisma/schema.prisma` with this content:

```prisma
generator client {
  provider = "prisma-client"
  output   = "../src/generated/prisma"
}

datasource db {
  provider = "postgresql"
}

model todo {
  id          Int       @id @default(autoincrement())
  title       String
  completedAt DateTime?
}
```

If `prisma.config.ts` was not generated by `npx prisma init`, create it manually in the project root:

```ts
import "dotenv/config";
import { defineConfig } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: {
    path: "prisma/migrations",
  },
  datasource: {
    url: process.env["POSTGRES_URL"],
  },
});
```

If you rename the database URL variable in `.env`, update this line to match the new variable name.

Generate Prisma Client:

```bash
npx prisma generate --schema prisma/schema.prisma
```

Create and apply your first migration (this creates the `todo` table):

```bash
npx prisma migrate dev --schema prisma/schema.prisma --name init
```

After that, Prisma will create SQL migration files inside `prisma/migrations` and apply them to your database.

## 5) Start the API

```bash
npm run dev
```

If everything is correct, you should see:

```txt
Server is running on port: 3000
```

## 6) Test endpoints

- Health: `GET http://localhost:3000/api/health`
- Todos: `GET http://localhost:3000/api/todos`

## Prisma commands explained

```bash
# generate Prisma Client from schema.prisma
npx prisma generate --schema prisma/schema.prisma

# create a new migration locally and apply it (development)
npx prisma migrate dev --schema prisma/schema.prisma --name <migration_name>

# apply existing migrations (production/staging)
npx prisma migrate deploy --schema prisma/schema.prisma

# sync schema directly to DB without creating migration files (quick prototyping)
npx prisma db push --schema prisma/schema.prisma
```

Recommended workflow:

1. Change `prisma/schema.prisma`
2. Run `npx prisma migrate dev --schema prisma/schema.prisma --name <migration_name>`
3. Commit `prisma/schema.prisma`
4. Commit `prisma/migrations/*`

## Fallback: manual SQL (only if Prisma fails in your environment)

If Prisma migration commands fail and you need a quick unblock, create the table manually:

```bash
docker exec -it restweb-postgres-db-1 psql -U postgres -d TODO -c 'CREATE TABLE IF NOT EXISTS public.todo (id SERIAL PRIMARY KEY, title TEXT NOT NULL, "completedAt" TIMESTAMP NULL);'
```

Verify:

```bash
docker exec -it restweb-postgres-db-1 psql -U postgres -d TODO -c '\dt public.*'
```

## Useful commands

```bash
# stop containers
docker compose down

# stop containers and remove postgres data (full reset)
docker compose down -v
```
